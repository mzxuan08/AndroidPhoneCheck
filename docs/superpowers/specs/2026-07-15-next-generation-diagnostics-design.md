# AndroidPhoneCheck 下一代验机设计（0.3.0）

## 目标

在现有 Android 9–16、纯手机端、离线验机流程上，提高屏幕覆盖率和检测结论的客观性。算法只提供辅助判断，最终仍允许商家人工确认。所有结论必须给出依据，不把环境问题误报成硬件故障。

## 设计原则

- 全程离线，不上传照片、录音和设备信息。
- 结果统一分为：正常、疑似异常、环境不适合、未测试、不支持、权限不足。
- 自动算法与人工确认并存；人工结论覆盖算法建议时记录两者。
- Android 9–16 使用同一套功能入口，系统受限项明确说明，不崩溃、不误判。
- 临时相机帧只在内存中分析；临时录音在完成或退出测试后删除。

## 1. 屏幕与触控

显示测试进入沉浸式全屏，隐藏状态栏、导航栏和应用操作栏，测试色覆盖可绘制区域。依次显示黑、白、红、绿、蓝和灰阶；单击切换颜色，双击显示或隐藏控制浮层，长按退出，避免控制按钮长期遮挡屏幕。退出时恢复系统栏。

触控测试同样使用沉浸式全屏。屏幕划分网格，手指经过后永久点亮对应格；同时绘制各触点轨迹、触点编号、当前触点数和本次最大触点数。完成条件是所有可触区域网格均被覆盖；刘海、圆角和系统手势保留区作为不可测区域说明，不判故障。

设备信息补充当前分辨率、像素密度、刷新率和可用显示模式。刷新率仅作为参数展示，不宣称能够检测面板是否原装。

## 2. 摄像头离线辅助检测

CameraX 预览继续支持前后摄像头切换，并增加 2–3 秒分析流程。分析器从连续低分辨率亮度帧计算：

- 拉普拉斯方差：辅助判断模糊或无法对焦；
- 亮度直方图：判断严重过曝、欠曝；
- RGB 通道均值差：提示明显偏色；
- 高频残差：提示异常噪点；
- 连续帧差：提示画面冻结；
- 低纹理与极端暗帧组合：提示镜头可能被遮挡。

阈值按帧尺寸和场景亮度归一化，并要求多帧持续满足才提示异常。场景太暗、过于单一或手机晃动过大时返回“环境不适合”，提示用户对准有纹理、光线正常的静止物体重测。算法输出每项分数、阈值和文字依据；用户查看实时画面后选择人工正常或异常。

这不是镜片划痕、坏点或原装摄像头鉴定，界面明确展示能力边界。

## 3. 麦克风与扬声器

音频测试分两段：约 2 秒环境底噪采样，再引导用户说话约 5 秒。离线计算：

- RMS 与峰值电平；
- 语音段相对底噪的信噪比；
- 接近满幅的削波比例；
- 突然归零或能量异常中断的断音比例；
- FFT 中 80–8,000 Hz 的能量分布，判断是否存在有效人声频段。

环境持续嘈杂、说话过轻或未检测到稳定语音时返回“环境不适合”，不直接判麦克风故障。界面显示实时波形、检测指标和建议，并保留录音回放供人工确认。

扬声器增加左声道、右声道和扫频测试音。由于手机扬声器位置和声道映射因机型而异，应用只验证音频能否播放并让用户确认是否清晰、有破音；不依据系统回录自动判定扬声器音质。

## 4. 引导式传感器检测

传感器页从“显示实时数值”升级为动作引导：

- 光线传感器：先记录环境值，再遮挡并移开，判断变化幅度；
- 距离传感器：引导靠近和离开听筒，判断是否出现近/远变化；
- 加速度计：摇动手机，判断三轴变化量；
- 陀螺仪：旋转手机，判断角速度变化；
- 磁力计：转动手机，判断三轴磁场是否响应；
- 其他气压、重力、温湿度等传感器：存在时显示实时数据和变化范围，不存在则标记不支持。

每项记录传感器名称、厂商、量程、分辨率、采样次数和检测到的变化范围。动作不足时提示重试；无传感器与传感器存在但无响应必须区分。

## 5. 网络与设备参数

连接页在已有 Wi‑Fi、蓝牙、NFC、定位状态基础上补充：

- Wi‑Fi 当前连接状态、RSSI 和信号等级；
- 移动网络制式（如 4G/5G）、SIM 是否存在及可读取的运营商信息；
- 蓝牙、NFC、定位的“支持/开启/权限受限”三态。

Android 新版本隐藏或限制的信息不通过非公开接口绕过。无权限、无 SIM、未开启和硬件不支持分别展示，均不误判为损坏。

## 6. 结果、数据流与异常处理

各检测模块输出统一的 `DiagnosticEvidence`：模块、指标名、实测值、参考阈值、算法建议、环境说明和时间。检测页面将证据提交给会话存储，汇总页展示人工结论、算法建议及关键数值。

相机和音频分析在后台线程执行，界面只接收采样进度和结果。性能不足时降低采样分辨率或帧率，并展示“已降低分析精度”；不阻塞预览。权限被拒绝、相机占用、录音初始化失败、传感器注销和页面中途退出都有明确状态，并确保释放 CameraX、AudioRecord、MediaPlayer 和传感器监听器。

## 7. 实现边界

本版本不加入云端 AI、机型真伪数据库、序列号联网查询、拆修历史推断或“是否原装”结论。文章中能确认的显示、多点触控、扬声器、麦克风、光线、加速度、震动、音量键、传感器、电池、网络、摄像头、GPS 和指纹项目纳入现有流程或本次增强；保存网页缺失的第 11–13 项不凭空实现。

## 8. 验收与测试

- 单元测试覆盖相机帧指标、音频指标、动态阈值、传感器变化判定和证据映射。
- Android 构建、单元测试和 lint 全部通过。
- 在现有中兴 Blade 20 Smart（Android 9）真机验证权限、全屏恢复、触控轨迹、相机切换、音频录放、传感器引导和页面退出资源释放。
- Android 10–16 无现有真机时，先通过 API 兼容分支测试与构建验证；以后接入设备再补真机记录。
- 验收标准：任何自动异常都有指标依据；环境不足不被记为硬件异常；所有测试均可人工完成并进入汇总报告。
