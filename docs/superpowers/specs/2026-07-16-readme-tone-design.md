# README 文案调整设计

## 目标

将公开仓库 README 改为普通开发者的朴素说明，减少宣传感、模板感和不必要的算法术语，让二手手机商家能快速看懂项目用途。

## 调整范围

- 开头直接说明项目用途、适用系统和离线特点。
- 功能按实际验机流程归类，不堆叠技术指标。
- 摄像头和麦克风只说明提供基础辅助判断，不使用“AI”“智能”等表述。
- 保留隐私、构建方法和能力边界。
- 明确项目仍需不同机型测试，检测结果需要人工复核。
- 不修改应用代码、版本号、GitHub Release 或其他文档。

## 文案风格

使用简短、事实性的中文；避免夸张形容词、营销口号和重复免责声明。README 仍保留必要的技术信息，方便开发者构建项目。
