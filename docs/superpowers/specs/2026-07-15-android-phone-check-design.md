# 安卓验机 App 设计说明

## 1. 目标

开发一款面向二手手机商家的纯手机端安卓验机 App，在不连接电脑、不 Root、无云端服务的前提下，对 Android 10–16 主流品牌手机进行尽可能全面且可信的功能检测和硬件信息采集。

首版应用暂名“安卓验机”，包名暂定为 `com.androidphonecheck.app`。项目位于 `F:\Code\AndroidPhoneCheck`。

## 2. 产品原则

- 完全离线运行，不要求登录，不上传验机数据。
- 自动检测与人工交互测试结合，优先保证结论可靠。
- 读取不到的数据不猜测，明确标记“不支持”或“权限不足”。
- 不用单一总分掩盖关键故障，以异常项目和严重程度为核心。
- 公共能力覆盖主流品牌，品牌差异通过独立适配层处理。

## 3. 支持范围

### 3.1 系统与品牌

- 最低系统：Android 10（API 29）
- 目标系统：Android 16（API 36）
- 主要品牌：小米、华为、荣耀、OPPO、vivo、三星及其他遵循 Android 公共 API 的设备
- 不保证每台设备支持所有项目；厂商未公开或系统禁止的能力应明确降级

### 3.2 验机项目

1. 基本信息：品牌、型号、Android 版本、CPU ABI、内存、存储容量与剩余空间、显示分辨率和刷新率。
2. 屏幕：纯色坏点、灰阶、亮度调节和漏光辅助观察。
3. 触摸：全屏划线、断触、多点触控和边缘触控。
4. 显示能力：分辨率、密度、HDR、可用刷新率和广色域能力。
5. 摄像头：前后摄及可枚举镜头、闪光灯、对焦、预览、拍照和录像。
6. 音频：听筒、扬声器、麦克风、录音回放及耳机或蓝牙音频辅助测试。
7. 实体功能：音量键、电源键辅助确认、震动和手电筒。
8. 传感器：加速度、陀螺仪、距离、光线、磁场、气压计等可用传感器及数据变化。
9. 连接能力：Wi-Fi、蓝牙、移动网络、NFC 和定位。
10. 电池与充电：电量、温度、电压、充电状态和系统可识别的充电类型。健康度只在平台可靠提供时显示。
11. 生物识别：检测指纹或人脸能力，并调起系统认证流程；不读取生物信息。
12. USB 与存储：USB 状态、OTG 能力辅助判断和应用私有目录读写测试。
13. 安全状态：Root、Bootloader、系统完整性和模拟器迹象，只作为风险提示，不作绝对结论。

### 3.3 不承诺项目

普通第三方 App 无法在所有 Android 10–16 设备上可靠获得或判断以下信息：IMEI、设备序列号、真实激活时间、维修史、进水记录、账号锁、是否换屏、是否换电池、存储扩容或改码。

只有存在公开、稳定且可验证的依据时才显示相关线索，否则标记“不支持”，不得输出推测性结论。

## 4. 验机流程

1. 首页提供“开始验机”和未完成验机入口。
2. 开始后采集基本信息和硬件能力清单。
3. 权限按测试项目即时申请。拒绝权限不阻断整体流程，对应项目标记“权限不足”。
4. 引导用户依次完成屏幕、触摸、摄像头、音频、按键、震动、传感器、网络和充电测试。
5. 每个项目可以重测或跳过；跳过项目记为“未测试”。
6. App 退出后保存进度，重新打开可以继续；用户可以清除记录并重新验机。
7. 完成后显示分类结果、异常数量、风险数量和未完成数量。

## 5. 结果模型

每个测试项使用以下状态之一：

- 正常：测试完成且结果符合预期。
- 异常：测试完成且出现明确故障，并记录具体现象。
- 风险提示：存在可疑线索，但证据不足以认定故障或改装。
- 未测试：用户跳过或测试没有完成。
- 不支持：系统、机型或硬件不提供该能力。
- 权限不足：用户未授予执行该测试所需的权限。

结果同时记录严重程度。触摸断点、摄像头无法启动、麦克风无输入和无法充电等关键问题应在总结页置顶。首版不计算百分制或笼统的合格分数。

## 6. 技术架构

- 语言：Kotlin
- UI：Jetpack Compose 与 Material 3
- 应用结构：单 Activity、MVVM
- 本地持久化：Room
- 并发：Kotlin Coroutines 与 Flow
- 模块边界：基础信息、显示触摸、摄像头、音频、传感器、连接、电池充电、安全风险、结果汇总
- 适配策略：公共检测接口加品牌适配实现；首版不依赖隐藏 API

每个检测模块暴露统一的能力探测、开始测试、停止测试和结果输出接口。UI 只消费统一结果模型，不直接依赖厂商实现。

## 7. 权限与隐私

- 仅在用户进入相应测试时申请摄像头、麦克风、蓝牙、附近设备和定位等权限。
- 权限拒绝、永久拒绝和系统不提供能力必须分别处理。
- 生物识别仅调用 Android 系统认证界面。
- 验机数据只存储在本机应用数据库中；首版无网络上传、账号、分享、PDF 或二维码功能。

## 8. 错误处理

- 平台 API 缺失、厂商返回空值、硬件不存在、权限拒绝及测试中断不得导致崩溃。
- 检测模块将底层异常转换为用户可理解的状态和原因。
- 相机、录音、传感器监听等资源在页面退出、测试结束或异常时及时释放。
- 自动检测结果与用户人工确认冲突时，保留原始自动结果和人工选择，并在汇总页提示复核。

## 9. 测试与验收

- 为结果模型、状态转换、严重程度排序、能力降级和数据持久化编写单元测试。
- 为权限拒绝、跳过、重测、退出恢复和结果汇总编写 Compose 界面测试。
- 建立 Android 10–16 的模拟器兼容测试矩阵。
- 使用小米、华为或荣耀、OPPO 或 vivo、三星真机执行人工验机清单；缺少真机时，对应品牌只能标记为尚未完成兼容性验证。
- 验收标准：流程可完整走通；任何单项失败不导致 App 崩溃；结果状态与实际执行情况一致；退出后能恢复进度；无网络也可使用全部首版功能。

## 10. 首版边界

首版聚焦完整流程、可靠的公共 API 检测和准确的能力降级。云端报告、账号体系、PDF、二维码、历史多设备管理、厂商付费查询接口及无法可靠验证的翻新判断均不在首版范围内。
