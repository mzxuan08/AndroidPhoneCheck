# AndroidPhoneCheck 0.3.0 实施计划

1. 扩展诊断领域模型，加入“环境不适合”“疑似异常”和结构化检测证据；升级持久化并兼容已有会话。
2. 重构显示与触控页面，加入可恢复的沉浸式系统栏控制、全屏色块、手势控制、多指轨迹和网格覆盖统计。
3. 建立纯 Kotlin 图像指标模块和单元测试；以 CameraX ImageAnalysis 接入连续帧采样，展示评分与环境判断。
4. 建立纯 Kotlin 音频指标模块和单元测试；升级录音流程为底噪/语音两阶段，增加波形、算法结果、自动清理和扬声器声道/扫频测试。
5. 将传感器实时列表升级为动作引导和变化量判定，保存传感器元数据与证据。
6. 补充刷新率、显示模式、Wi-Fi RSSI、移动网络和 SIM 可用信息，并处理 Android 9–16 权限/API 差异。
7. 在汇总页展示算法建议、人工结论和关键证据；升级版本至 0.3.0。
8. 运行单元测试、debug 构建和 lint；安装到 Android 9 中兴真机并检查崩溃、权限、资源释放和关键流程。

每个阶段保持可构建，功能按模块提交，避免一次性大改导致无法定位回归。
